# X 长线程母稿：Polanyi 悖论，为什么决定了 AI skill 的上限

## 推荐标题

- Polanyi 悖论，几乎解释了 AI skill 为什么总差最后一步
- 为什么 AI 会做很多事，却总在关键一步露怯
- 想把 tacit knowledge 做成 agent，先过 Polanyi 这一关

## 版本说明

这是一版适合发 X 的长线程母稿。核心目标有三个：

- 讲清楚 Polanyi 悖论和 AI skill learning 的关系
- 把 `Polanyi Stack` 放进一个更大的叙事里
- 让读者自然过渡到 repo，而不是看完觉得你只是在推链接

## 线程正文

**1/**

我越来越觉得，Polanyi 悖论几乎解释了 AI skill 为什么总差最后一步。

模型能看文档，能学格式，能模仿答案。

真正难的部分，常常藏在一句话里：

**We know more than we can tell.**

我们知道的，往往比我们能完整说出的多得多。

**2/**

这句话听起来像哲学。

落到 AI 上，意思很硬：

很多高价值能力，并不以规则、流程、条款的形式存在。

它们活在：

- 手感
- 时机
- 例外处理
- 风险边界
- 哪个信号值得盯，哪个信号该忽略

这层东西，人靠经验长出来，模型很难靠说明书直接拿到。

**3/**

所以很多 AI 系统会出现一个很熟悉的现象：

看起来什么都懂，
真正上场总差一点。

文档能复述。
流程能跑。
案例能模仿。

一到现场变化、边界情况、分布外问题，就开始露怯。

**4/**

这也是为什么很多人做 agent，做到最后会有一种挫败感：

prompt 已经很长了，
约束也加了，
示例也堆了，
输出看起来也挺像样。

结果还是不稳。

原因通常不神秘。

你写进去的，只是显性知识。
真正拉开差距的判断层，还没被提出来。

**5/**

Polanyi 悖论真正击中的，是自动化里一个很尴尬的前提：

软件喜欢清晰规则。
高手表现里最值钱的部分，经常没有清晰规则。

或者说，规则有，但它们散在经验里，没被压缩成能直接复用的形式。

这就是为什么“把高手的方法写成 SOP”这件事，经常写着写着就空了。

**6/**

你去看很多难自动化的工作，结构都很像：

- 代码 review
- 带新人
- 临场诊断
- 设计评审
- 销售判断
- 内容选题

共同点不是“复杂”。

共同点是：决定质量的那一层，往往在 tacit knowledge。

**7/**

比如代码 review。

一个强 reviewer 看 PR，不只是看语法和规范。

他会突然皱一下眉，因为某个地方“味道不对”。

可能是信任边界。
可能是隐蔽副作用。
可能是结构已经开始往坏方向长。

这类判断，事后能解释一点。
现场做判断时，很多是半显性的。

**8/**

再比如带新人。

文档都给了，流程也讲了，新人还是做不出来。

问题通常不在“没学过”。

问题在于：

他不知道什么时候该慢下来，
不知道哪几个信号最关键，
不知道什么情况该自己继续做，
什么情况必须立刻升级。

这些全是 tacit knowledge。

**9/**

所以我现在对 AI skill 的看法越来越明确：

**Skill 文件写的是骨架。**

骨架包括：

- 目标
- 触发条件
- 步骤
- 输出格式
- 边界

真正让系统长出判断的，很多时候还得靠：

- 示例
- 反例
- 观察
- 模仿
- 复盘
- 反馈闭环

**10/**

这也是 Polanyi 悖论对 AI skill design 最有价值的地方。

它不是让你悲观，
它是帮你看清边界。

哪些东西可以写进 skill。
哪些东西要靠案例去逼近。
哪些东西要靠真实交互和反复校正。
哪些东西今天就不该过度承诺自动化。

边界看清了，系统反而更能打。

**11/**

很多人会问：

那是不是说明 AI 永远学不会 tacit knowledge？

我觉得结论没必要走那么极端。

更接近现实的说法是：

AI 已经能从大量示例里吸收一部分隐性模式，
但凡涉及现场判断、适应性、创造性取舍、人际互动，
挑战依然很大。

悖论没有消失。
战场换了地方。

**12/**

过去的难点是：

规则写不全。

今天的难点更像是：

- 表示学得对不对
- 泛化稳不稳
- 边界感够不够
- 分布外场景会不会崩

说白了，问题从“写规则”变成了“长判断”。

**13/**

这也是我做这个 repo 的原因：

**Polanyi Stack**

它想解决的不是“怎么把 prompt 写得更花”。

它想解决的是：

怎么把高手脑子里那层难讲清的东西，
尽量拆出来，
再落成可复用的 agent、skill 和训练系统。

repo 在这：

https://github.com/11Yuxuanyang/polanyi-stack

**14/**

我现在把它理解成一套给 AI builders 的 Polanyi stack。

核心几件事：

- 提取 tacit cues
- 设计练习梯度
- 做个人知识地图
- 设计师徒传承
- 训练 attention switching

这几件事连起来，才更接近“把能力产品化”。

**15/**

如果你也在做 agent，或者在想怎么把专家判断沉淀成系统，
我建议你盯住一个问题：

**你现在交给模型的，到底是知识，还是判断？**

如果只是知识，文档和 prompt 往往够用。

如果真正难点在判断，那你迟早会走到 Polanyi 这里。

## 结尾版本

### 结尾 A：偏思想传播

AI 这一轮最有意思的地方，可能不是模型会了多少，
而是我们终于被迫认真面对一个老问题：

人类最值钱的能力，到底有多少是可以直接说清楚的。

### 结尾 B：偏 repo 导流

我把这套东西整理成了一个开源 repo，叫 `Polanyi Stack`。

里面不是空谈 tacit knowledge，
而是直接把它往 agent、skill、onboarding 和训练系统上落。

repo:
https://github.com/11Yuxuanyang/polanyi-stack

文档站：
https://11yuxuanyang.github.io/polanyi-stack/

### 结尾 C：偏互动

你做过最难写成规则、却又明显影响结果的能力，是什么？

代码 review？
带人？
销售？
创作判断？

我怀疑很多真正值钱的能力，都卡在这里。
